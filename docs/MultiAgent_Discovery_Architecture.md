# Multi-Agent Discovery OS Architecture Plan
**Status:** Design Phase - DO NOT IMPLEMENT YET
**Date:** 2026-03-02
**Version:** 1.0

---

## Executive Summary

This document specifies a **multi-agentic Discovery OS** where:
1. A **large orchestrator model** (MiniMax 2.5 / GPT-4 class) understands complex research goals and deploys specialist agents
2. **Specialist agents** write deterministic Python scripts that make tool calls (RDKit, external APIs)
3. Researchers **see active agents** in real-time with full transparency
4. All artifacts (scripts, logs, results) are saved to the session folder for reproducibility
5. The system integrates seamlessly with existing backend (LangGraph + FastAPI) and frontend (React + Zustand)

---

## Current State Analysis

### ✅ What's Already Built

#### Backend Infrastructure
- **Session Management** (`discovery_session.py`): Creates `discovery/{session_id}/` folders, stores sessions in SQLite
- **Coordinator Agent** (`coordinator.py`): HITL goal extraction using LangGraph `interrupt()`, thread ID `coordinator-{session_id}`
- **Discovery ReAct Agent** (`discovery_graph.py`): Single-agent think→execute loop, uses RDKit plugins
- **Executor Agent** (`executor.py`): Script generation/approval/execution scaffold (not fully integrated)
- **Plugin System** (`plugins/`): BasePlugin ABC, PluginManager singleton, RDKit tools
- **LLM Service** (`llm.py`): Constrained JSON generation (GBNF + `response_format`), chat template formatting
- **Memory** (`app.core.memory`): Shared MemorySaver singleton for LangGraph checkpointing
- **SSE Streaming** (`routes.py`): StreamingResponse with event generators + cancel events

#### Frontend Infrastructure
- **DiscoveryStore** (`discoveryStore.ts`): Multi-session, epoch-based branching, stage progression (1-7)
- **ChatStore** (`chatStore.ts`): Threaded conversations, multi-mode support (librarian/cortex/moe/discovery/coordinator)
- **DiscoveryWorkbench** (`DiscoveryWorkbench.tsx`): Telemetry panel, ReAct trace viewer, candidate table
- **DiscoveryWorkspaceTab** (`DiscoveryWorkspaceTab.tsx`): Coordinator chat + workbench split view, session files panel
- **SSE Stream Adapter** (`stream-adapter.ts`): Normalized SSE event parsing

#### Gaps & Limitations
1. **No orchestrator**: Discovery graph is single-agent ReAct, not multi-agent
2. **No agent registry**: No tracking of active agents or their states
3. **No agent visibility**: Researcher can't see which agents are running or what they're doing
4. **Executor not integrated**: Script generation exists but doesn't tie into orchestrator workflow
5. **No parallel agents**: Current system is strictly sequential
6. **No dynamic agent spawning**: Agents are hardcoded, not spawned based on task analysis

```python
class AgentStatus(str, Enum):
    SPAWNED = "spawned"        # Created but not started
    RUNNING = "running"        # Actively executing
    WAITING = "waiting"        # Blocked on HITL or dependency
    COMPLETED = "completed"    # Finished successfully
    FAILED = "failed"          # Encountered error
    CANCELLED = "cancelled"    # User or orchestrator cancelled
```

#### Agent Registry (Backend)
```python
# src/backend/app/services/agent_registry.py

from typing import Dict, List, Optional
from datetime import datetime
from pydantic import BaseModel

class AgentMetadata(BaseModel):
    agent_id: str                    # script_writer_001
    agent_type: str                  # "script_writer" | "literature" | "validator"
    session_id: str
    status: AgentStatus
    task_description: str            # "Generate SMILES enumeration script"
    spawned_at: datetime
    completed_at: Optional[datetime]
    parent_agent_id: Optional[str]   # For hierarchical spawning
    artifacts: List[str]             # Paths to generated files
    progress: float                  # 0.0 - 1.0
    current_step: str                # Human-readable current action
    thread_id: str                   # LangGraph thread ID
    dependencies: List[str]          # Agent IDs this depends on
    error_message: Optional[str]

class AgentRegistry:
    """Singleton registry of active agents in all sessions."""

    def __init__(self):
        self._agents: Dict[str, AgentMetadata] = {}

    def register_agent(self, metadata: AgentMetadata) -> None:
        """Register a newly spawned agent."""
        self._agents[metadata.agent_id] = metadata

    def update_status(self, agent_id: str, status: AgentStatus, **kwargs) -> None:
        """Update agent status and optional fields."""
        if agent_id in self._agents:
            self._agents[agent_id].status = status
            for key, value in kwargs.items():
                setattr(self._agents[agent_id], key, value)

    def get_session_agents(self, session_id: str) -> List[AgentMetadata]:
        """Get all agents for a session, sorted by spawn time."""
        return sorted(
            [a for a in self._agents.values() if a.session_id == session_id],
            key=lambda a: a.spawned_at
        )

    def get_active_agents(self, session_id: str) -> List[AgentMetadata]:
        """Get currently active agents (spawned/running/waiting)."""
        return [
            a for a in self.get_session_agents(session_id)
            if a.status in (AgentStatus.SPAWNED, AgentStatus.RUNNING, AgentStatus.WAITING)
        ]

# Singleton instance
_registry: Optional[AgentRegistry] = None

def get_agent_registry() -> AgentRegistry:
    global _registry
    if _registry is None:
        _registry = AgentRegistry()
    return _registry
```

### 3. Orchestrator Agent Implementation

The orchestrator replaces the single-agent discovery_graph.py. It uses a large model for planning and agent coordination.

#### Orchestrator Graph
```python
# src/backend/app/services/agents/orchestrator.py

from typing import TypedDict, List, Dict, Any, Optional
from langgraph.graph import StateGraph, END
from app.services.llm import LLMService
from app.services.agent_registry import get_agent_registry, AgentMetadata, AgentStatus
import asyncio

class OrchestratorState(TypedDict, total=False):
    session_id: str
    project_id: str
    research_goals: List[str]           # From coordinator
    target_params: Dict[str, Any]       # From coordinator

    # Task decomposition
    task_plan: List[Dict[str, Any]]     # [{task, agent_type, priority, dependencies}]
    completed_tasks: List[str]
    current_phase: str                  # "planning" | "execution" | "synthesis"

    # Agent management
    active_agents: List[str]            # Agent IDs currently running
    agent_results: Dict[str, Any]       # {agent_id: result}

    # Global state
    iteration: int
    max_iterations: int
    status: str                         # "running" | "complete" | "failed"
    final_output: Optional[Dict[str, Any]]

# Constrained generation schema for task planning
TASK_PLANNING_SCHEMA = {
    "type": "object",
    "properties": {
        "analysis": {"type": "string", "description": "Assessment of research goals"},
        "tasks": {
            "type": "array",
            "items": {
                "type": "object",
                "properties": {
                    "task_id": {"type": "string"},
                    "description": {"type": "string"},
                    "agent_type": {
                        "type": "string",
                        "enum": ["script_writer", "literature", "property_predictor", "validator", "synthesis_planner", "analysis"]
                    },
                    "priority": {"type": "integer", "minimum": 1, "maximum": 10},
                    "dependencies": {"type": "array", "items": {"type": "string"}},
                    "parallel": {"type": "boolean"}
                },
                "required": ["task_id", "description", "agent_type", "priority"]
            }
        },
        "execution_strategy": {
            "type": "string",
            "enum": ["parallel", "sequential", "hybrid"],
            "description": "How to execute tasks"
        }
    },
    "required": ["analysis", "tasks", "execution_strategy"]
}

def _build_orchestrator_graph(llm_service: LLMService) -> StateGraph:
    """Build orchestrator StateGraph that spawns and manages specialist agents."""

    async def plan_tasks(state: OrchestratorState) -> dict:
        """Node 1: Analyze goals and create task decomposition."""
        goals = state.get("research_goals", [])
        params = state.get("target_params", {})

        # Check existing artifacts from previous runs
        session_id = state["session_id"]
        registry = get_agent_registry()
        completed = [a for a in registry.get_session_agents(session_id) if a.status == AgentStatus.COMPLETED]

        context = f"""You are the orchestrator for a scientific discovery session.

RESEARCH GOALS:
{chr(10).join(f"- {g}" for g in goals)}

TARGET PARAMETERS:
{params}

COMPLETED AGENTS SO FAR:
{chr(10).join(f"- {a.agent_type}: {a.task_description} → {len(a.artifacts)} artifacts" for a in completed) if completed else "None yet"}

AVAILABLE AGENT TYPES:
- script_writer: Generates Python scripts using RDKit, pandas, NumPy for data generation/analysis
- literature: Searches corpus for relevant context, extracts citations
- property_predictor: Coordinates ML property prediction tools (MW, LogP, toxicity, etc.)
- validator: Verifies outputs against constraints, checks for errors
- synthesis_planner: Plans synthetic routes using retrosynthesis tools
- analysis: Interprets results, generates summaries and insights

RESOURCE CONSTRAINTS:
- 4GB VRAM available (sequential execution preferred for heavy ML tasks)
- Local execution (no cloud APIs requiring auth)

Your task: Decompose the research goals into specific agent tasks. For each task:
1. Choose the appropriate agent type
2. Set priority (1=highest, 10=lowest)
3. Specify dependencies (which tasks must complete first)
4. Mark if tasks can run in parallel

Respond with the required JSON schema."""

        try:
            plan = await llm_service.generate_constrained(
                prompt=context,
                schema=TASK_PLANNING_SCHEMA,
                temperature=0.3,
                max_tokens=2048,
            )
        except Exception as e:
            logger.error(f"Orchestrator planning failed: {e}")
            return {"status": "failed", "current_phase": "planning"}

        tasks = plan.get("tasks", [])
        strategy = plan.get("execution_strategy", "sequential")

        return {
            "task_plan": tasks,
            "current_phase": "execution",
            "execution_strategy": strategy,
        }

    async def spawn_agents(state: OrchestratorState) -> dict:
        """Node 2: Spawn specialist agents based on task plan."""
        task_plan = state.get("task_plan", [])
        completed_tasks = set(state.get("completed_tasks", []))
        agent_results = state.get("agent_results", {})
        session_id = state["session_id"]
        registry = get_agent_registry()

        # Find tasks ready to execute (dependencies satisfied)
        ready_tasks = []
        for task in task_plan:
            task_id = task["task_id"]
            if task_id in completed_tasks:
                continue

            deps = set(task.get("dependencies", []))
            if deps.issubset(completed_tasks):
                ready_tasks.append(task)

        if not ready_tasks:
            # All tasks complete or waiting on dependencies
            return {"current_phase": "synthesis"}

        # Spawn agents for ready tasks
        spawned = []
        for task in ready_tasks:
            agent_id = f"{task['agent_type']}_{task['task_id']}"

            # Register agent
            metadata = AgentMetadata(
                agent_id=agent_id,
                agent_type=task["agent_type"],
                session_id=session_id,
                status=AgentStatus.SPAWNED,
                task_description=task["description"],
                spawned_at=datetime.now(),
                thread_id=f"{agent_id}-{session_id}",
                dependencies=task.get("dependencies", []),
                artifacts=[],
                progress=0.0,
                current_step="Initializing...",
            )
            registry.register_agent(metadata)

            # Spawn agent execution (async task)
            asyncio.create_task(_execute_specialist_agent(
                agent_id=agent_id,
                agent_type=task["agent_type"],
                task=task,
                state=state,
                llm_service=llm_service,
            ))

            spawned.append(agent_id)

        return {
            "active_agents": spawned,
            "current_phase": "execution",
        }

    async def monitor_agents(state: OrchestratorState) -> dict:
        """Node 3: Wait for active agents to complete."""
        session_id = state["session_id"]
        registry = get_agent_registry()

        # Poll until all agents complete
        max_wait = 600  # 10 minutes
        poll_interval = 2  # seconds
        waited = 0

        while waited < max_wait:
            active = registry.get_active_agents(session_id)

            if not active:
                # All agents done
                break

            await asyncio.sleep(poll_interval)
            waited += poll_interval

        # Collect results
        all_agents = registry.get_session_agents(session_id)
        completed = [a for a in all_agents if a.status == AgentStatus.COMPLETED]
        failed = [a for a in all_agents if a.status == AgentStatus.FAILED]

        results = {}
        for agent in completed:
            results[agent.agent_id] = {
                "artifacts": agent.artifacts,
                "task": agent.task_description,
            }

        # Mark tasks as completed
        completed_task_ids = state.get("completed_tasks", [])
        for agent in completed:
            # Extract task_id from agent_id (format: {type}_{task_id})
            task_id = agent.agent_id.split("_", 1)[1]
            if task_id not in completed_task_ids:
                completed_task_ids.append(task_id)

        if failed:
            return {
                "status": "failed",
                "agent_results": results,
                "completed_tasks": completed_task_ids,
            }

        return {
            "agent_results": results,
            "completed_tasks": completed_task_ids,
            "iteration": state.get("iteration", 0) + 1,
        }

    async def synthesize_results(state: OrchestratorState) -> dict:
        """Node 4: Combine agent outputs into final result."""
        results = state.get("agent_results", {})
        goals = state.get("research_goals", [])

        # Aggregate all artifacts
        all_artifacts = []
        for agent_id, result in results.items():
            all_artifacts.extend(result["artifacts"])

        # Use LLM to synthesize findings
        summary_prompt = f"""Synthesize the results from the multi-agent discovery session.

ORIGINAL GOALS:
{chr(10).join(f"- {g}" for g in goals)}

AGENT RESULTS:
{chr(10).join(f"- {agent_id}: {result['task']} → {len(result['artifacts'])} artifacts" for agent_id, result in results.items())}

ARTIFACTS GENERATED:
{chr(10).join(f"- {a}" for a in all_artifacts)}

Provide a concise summary of what was accomplished and how it addresses the research goals."""

        summary = await llm_service.generate(
            prompt=summary_prompt,
            temperature=0.3,
            max_tokens=1024,
        )

        return {
            "status": "complete",
            "final_output": {
                "summary": summary,
                "artifacts": all_artifacts,
                "agent_count": len(results),
            },
        }

    def should_continue(state: OrchestratorState) -> str:
        """Routing logic."""
        phase = state.get("current_phase", "planning")
        iteration = state.get("iteration", 0)
        max_iter = state.get("max_iterations", 10)
        status = state.get("status", "running")

        if status in ("complete", "failed") or iteration >= max_iter:
            return "end"

        if phase == "planning":
            return "spawn"
        elif phase == "execution":
            return "monitor"
        elif phase == "synthesis":
            return "synthesize"

        return "spawn"

    # Build graph
    graph = StateGraph(OrchestratorState)

    graph.add_node("plan", plan_tasks)
    graph.add_node("spawn", spawn_agents)
    graph.add_node("monitor", monitor_agents)
    graph.add_node("synthesize", synthesize_results)

    graph.set_entry_point("plan")

    graph.add_conditional_edges("plan", should_continue, {
        "spawn": "spawn",
        "end": END,
    })

    graph.add_conditional_edges("spawn", should_continue, {
        "monitor": "monitor",
        "synthesize": "synthesize",
        "end": END,
    })

    graph.add_conditional_edges("monitor", should_continue, {
        "spawn": "spawn",
        "synthesize": "synthesize",
        "end": END,
    })

    graph.add_edge("synthesize", END)

    return graph

async def _execute_specialist_agent(
    agent_id: str,
    agent_type: str,
    task: Dict[str, Any],
    state: OrchestratorState,
    llm_service: LLMService,
) -> None:
    """Execute a specialist agent in the background."""
    registry = get_agent_registry()
    registry.update_status(agent_id, AgentStatus.RUNNING, current_step="Starting...")

    try:
        # Dispatch to appropriate agent implementation
        if agent_type == "script_writer":
            from app.services.agents.specialist_agents.script_writer import run_script_writer
            result = await run_script_writer(agent_id, task, state, llm_service)
        elif agent_type == "literature":
            from app.services.agents.specialist_agents.literature import run_literature_agent
            result = await run_literature_agent(agent_id, task, state, llm_service)
        elif agent_type == "property_predictor":
            from app.services.agents.specialist_agents.property_predictor import run_property_predictor
            result = await run_property_predictor(agent_id, task, state, llm_service)
        elif agent_type == "validator":
            from app.services.agents.specialist_agents.validator import run_validator
            result = await run_validator(agent_id, task, state, llm_service)
        else:
            raise ValueError(f"Unknown agent type: {agent_type}")

        registry.update_status(
            agent_id,
            AgentStatus.COMPLETED,
            completed_at=datetime.now(),
            artifacts=result.get("artifacts", []),
            progress=1.0,
        )

    except Exception as e:
        logger.exception(f"Agent {agent_id} failed")
        registry.update_status(
            agent_id,
            AgentStatus.FAILED,
            error_message=str(e),
            completed_at=datetime.now(),
        )
```

```python
# src/backend/app/api/routes.py

@router.post("/api/discovery/{session_id}/orchestrator/start")
async def start_orchestrator(
    session_id: str,
    request: Request,
):
    """Start the orchestrator agent for a discovery session.

    Requires coordinator to have already extracted goals.
    """
    # Get extracted goals from session
    session = DiscoverySessionService.get_session(session_id)
    goals = session.target_params.get("coordinator_extracted_goals", [])

    if not goals:
        raise HTTPException(
            status_code=400,
            detail="Session must complete coordinator phase first"
        )

    async def event_generator():
        cancel_event = asyncio.Event()
        monitor_task = asyncio.create_task(monitor_disconnect(request, cancel_event))

        try:
            from app.services.agents.orchestrator import run_orchestrator_streaming

            async for event_type, event_data in run_orchestrator_streaming(
                session_id=session_id,
                research_goals=goals,
                target_params=session.target_params,
                llm_service=get_llm_service(),
                cancel_event=cancel_event,
            ):
                yield f"event: {event_type}\n"
                yield f"data: {json.dumps(event_data)}\n\n"

        finally:
            monitor_task.cancel()

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
    )
```

# src/backend/app/services/agents/specialist_agents/script_writer.py

async def run_script_writer(
    agent_id: str,
    task: Dict[str, Any],
    state: OrchestratorState,
    llm_service: LLMService,
) -> Dict[str, Any]:
    """Script Writer Agent: Generates Python scripts for data generation/analysis."""

    registry = get_agent_registry()
    session_id = state["session_id"]

    # Create agent workspace
    agent_dir = Path(settings.DATA_DIR) / "discovery" / session_id / "agents" / agent_id
    agent_dir.mkdir(parents=True, exist_ok=True)

    registry.update_status(agent_id, AgentStatus.RUNNING, current_step="Analyzing task...")

    # Step 1: Plan script
    task_desc = task["description"]
    goals = state.get("research_goals", [])

    planning_prompt = f"""You are a scientific computing expert. Generate a Python script.

TASK: {task_desc}

RESEARCH CONTEXT:
{chr(10).join(f"- {g}" for g in goals)}

REQUIREMENTS:
- Use RDKit, pandas, NumPy (no external APIs requiring auth)
- Save all outputs to current directory
- Include error handling and logging
- Write to CSV/JSON for machine readability
- Print progress to stdout

Generate a complete, executable Python script."""

    registry.update_status(agent_id, AgentStatus.RUNNING, current_step="Generating script...", progress=0.3)

    script_code = await llm_service.generate(
        prompt=planning_prompt,
        temperature=0.4,
        max_tokens=2048,
    )

    # Clean code blocks
    if "```python" in script_code:
        script_code = script_code.split("```python")[1].split("```")[0].strip()

    # Save script
    script_path = agent_dir / f"{agent_id}.py"
    script_path.write_text(script_code, encoding="utf-8")

    registry.update_status(agent_id, AgentStatus.RUNNING, current_step="Executing script...", progress=0.6)

    # Execute script in sandbox
    result = subprocess.run(
        [sys.executable, str(script_path)],
        cwd=str(agent_dir),
        capture_output=True,
        text=True,
        timeout=300,
    )

    # Save execution log
    log_path = agent_dir / "execution_log.txt"
    log_path.write_text(
        f"Exit Code: {result.returncode}\n\nSTDOUT:\n{result.stdout}\n\nSTDERR:\n{result.stderr}",
        encoding="utf-8"
    )

    if result.returncode != 0:
        raise RuntimeError(f"Script execution failed: {result.stderr}")

    # Collect artifacts
    artifacts = [str(f.relative_to(agent_dir)) for f in agent_dir.iterdir() if f.is_file()]

    registry.update_status(agent_id, AgentStatus.RUNNING, current_step="Complete", progress=1.0)

    return {
        "artifacts": artifacts,
        "script_path": str(script_path),
        "execution_log": str(log_path),
    }

## Integration Workflow

### Phase 1: Coordinator → Orchestrator
1. User completes Mission Control (sets target params)
2. Coordinator agent runs, extracts goals via HITL
3. Goals saved to `DiscoverySession.target_params["coordinator_extracted_goals"]`
4. User clicks "Start Execution" → triggers orchestrator

### Phase 2: Orchestrator → Specialist Agents
1. Orchestrator analyzes goals using large model (MiniMax 2.5)
2. Decomposes into tasks: `[{task_id, agent_type, description, dependencies}]`
3. Spawns specialist agents dynamically
4. Registers each agent in AgentRegistry
5. Frontend polls `/api/discovery/{session_id}/agents` every 1s → updates AgentActivityPanel

### Phase 3: Specialist Execution
1. Each specialist agent:
   - Creates workspace: `discovery/{session_id}/agents/{agent_id}/`
   - Updates registry with progress
   - Generates artifacts (scripts, CSVs, logs)
   - Marks self as COMPLETED or FAILED
2. Orchestrator monitors via registry
3. When dependencies satisfied, spawns next batch

### Phase 4: Synthesis
1. All agents complete
2. Orchestrator aggregates results
3. Generates summary
4. Updates discoveryStore with final artifacts
5. Frontend displays results in workbench

## SOTA Patterns Applied

### 1. **Hierarchical Agent Spawning** (from AutoGen/CrewAI)
- Orchestrator spawns specialists
- Each specialist can spawn sub-agents if needed
- Parent-child relationships tracked via `parent_agent_id`

### 2. **Event-Driven Communication** (from LangGraph Multi-Agent)
- SSE events for orchestrator state changes: `orchestrator_planning`, `agent_spawned`, `agent_completed`
- Frontend subscribes to event stream
- Real-time UI updates without polling

### 3. **Resource-Aware Scheduling** (from Microsoft Magentic-One)
- Orchestrator knows about 4GB VRAM constraint
- Can choose sequential execution for heavy ML tasks
- Parallel execution for I/O-bound tasks (literature search, script generation)

### 4. **Artifact-Based Communication** (from Devin/Cursor)
- Agents don't communicate via messages
- They write artifacts to shared session folder
- Other agents read artifacts as inputs
- Fully transparent and auditable

### 5. **Human-in-the-Loop Checkpoints** (from GitHub Copilot Workspace)
- Critical steps pause for approval (script execution, API calls)
- Uses LangGraph `interrupt()` pattern
- Researcher can edit artifacts before execution resumes

## Benefits of This Architecture

### For Researchers
1. **Transparency**: See exactly which agents are running and what they're doing
2. **Control**: Approve/reject scripts before execution
3. **Reproducibility**: All artifacts saved with timestamps and provenance
4. **Flexibility**: Orchestrator adapts to different research domains

### For System
1. **Scalability**: Add new specialist agents without changing orchestrator
2. **Resource efficiency**: Sequential execution when needed for VRAM constraints
3. **Error isolation**: Agent failures don't crash entire session
4. **Modularity**: Each agent is independent and testable

### For Development
1. **Clear separation**: Orchestrator (planning) vs specialists (execution)
2. **Reusable components**: Specialist agents work across different research goals
3. **Easy debugging**: Agent logs + registry state provide full audit trail
4. **Incremental rollout**: Can deploy agents one at a time
